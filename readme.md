# 《竞拍之王》实时估值决策辅助系统 - 开发架构说明书

## 1. 项目概述与核心目标
本项目旨在开发一个面向游戏《竞拍之王》的**本地实时估值辅助系统**。游戏核心机制为多轮暗标竞价，玩家在信息不全的情况下对集装箱内藏品进行估值。
本系统**不使用端到端深度学习**（因为游戏规则存在严格的硬性数学约束，神经网络处理此类浮点、求余逻辑极易产生幻觉且效率低下）。系统采用 **混合架构 (Neuro-Symbolic)**：基于深度优先搜索 (DFS) 的**约束满足求解器 (CSP)** 结合基于 Numpy 矩阵运算的**蒙特卡洛模拟器 (Monte Carlo Simulator)**，最终通过 **Streamlit** 提供实时交互 GUI。

---

## 2. 游戏规则与数学建模 (Game Mechanics & Math Models)

集装箱内的藏品分为 6 个等级：白(1)、绿(2)、蓝(3)、紫(4)、橙(5)、红(6)。系统在 5 轮中逐步揭示信息。

### 2.1 基础变量定义
设各等级物品数量分别为 $N_W, N_G, N_B, N_P, N_O, N_R$。

基于经验，各等级物品从低到高的出现先验概率（权重）大致为：
$$W_W:W_G:W_B:W_P:W_O:W_R = 16:16:8:4:2:1$$

### 2.2 轮次信息与数学约束转化
* **第 1 轮 (总数约束):**
    * 线索：获得本局总数量 $N_{total}$。
    * 方程：$N_R + N_O + N_P + N_B + N_G + N_W = N_{total}$
* **第 2, 3, 4 轮 (特定等级约束 + 附加随机线索):**
    * 线索：分别获得橙色、紫色、蓝色藏品的**平均格数**（展示为小数点后至多两位的浮点数）。
    * 推导（同余约束）：将平均格数 $G_{avg}$ 转化为最简分数 $\frac{Numerator}{Denominator}$。该等级的物品数量 $N_x$ 必须满足：
        $$N_x \pmod{Denominator} = 0$$
* **第 5 轮 (低级物品总数约束):**
    * 线索：获得绿色和白色藏品的总数量 $N_{GW}$。
    * 方程：$N_G + N_W = N_{GW}$

### 2.3 任意附加线索的数学映射
在第 2、3、4 轮中，可能出现以下附加线索，需在代码中实现对应的过滤逻辑：
1.  **展示具体藏品/等级：** 设定下限。例如展示 1 件特定紫装，则 $N_P \ge 1$，且模拟池中必须包含该特定物品。
2.  **展示藏品轮廓：** 基于轮廓字典，设定特定候选物品集合的存在性约束。
3.  **某等级总格数 $G_{total}$：** 背包问题约束。选取该等级物品体积之和必须精确等于 $G_{total}$。
4.  **均价线索 (极度重要，精度控制)：** 提示某几件或某等级均价 $P_{avg}$。为了避免浮点数精度丢失，必须乘以 100 转化为整数运算。设对应物品总价为 $P_{total}$，数量为 $N_{sub}$：
    $$\lfloor (P_{avg} - 0.005) \cdot 100 \cdot N_{sub} \rfloor \le P_{total} \cdot 100 \le \lceil (P_{avg} + 0.005) \cdot 100 \cdot N_{sub} \rceil$$

## 3. 系统核心架构与模块划分

### 模块 A: 内存数据加载器 (Data Loader)
* **输入：** `items_data.csv`
* **字段要求：** `Item_ID` (int), `Item_Name` (str), `Grade` (int, 1-6), `Price` (float), `Grid_Size` (int)。
* **要求：** 在应用启动时一次性加载为 Pandas DataFrame 或 Python 字典，用于极速索引。

### 模块 B: 高并发约束求解引擎 (The Solver Engine)
* **核心逻辑：** 接收当前轮次录入的所有线索。利用 DFS (深度优先搜索) 或 `itertools` 生成所有可能的 $(N_W, N_G, N_B, N_P, N_O, N_R)$ 组合。
* **剪枝策略：** 将上述数学建模中的条件作为过滤器。遇到不满足同余方程、总数方程、均价不等式的组合直接丢弃。
* **输出：** 一个包含所有合法数量组合的列表 `valid_combinations`。

### 模块 C: 蒙特卡洛估值模拟器 (The Monte Carlo Simulator)
* **关于“训练方法”的说明：本系统无需前置训练模型。** 估值通过实时大规模随机采样完成。
* **核心逻辑：** 针对模块 B 输出的每个合法组合，结合 $16:16:8:4:2:1$ 的等级先验概率，在对应等级的物品池中随机抽取具体物品。
* **性能要求：** 使用 Numpy 矩阵化运算。为每个合法组合生成 10,000 个虚拟集装箱，计算总价。
* **输出：**
    * `Expected_Value` (期望均值)
    * `Safe_Bid_95` (95% 置信区间下限，用于保守出价)
    * `Max_Value` (理论最大值)

### 模块 D: Streamlit 交互界面 (GUI)
* **左侧边栏 (Sidebar)：** 状态指示灯（CSV 加载状态），当前轮次选择器，线索录入表单（下拉框选择线索类型，输入框填人数值），“添加线索”按钮，“重置本局”按钮。
* **右侧主屏 (Main Panel)：** 三个 Metric 大屏展示模块 C 输出的三个价格。显示模块 B 当前剩余的“可能组合数量”（漏斗状态）。
* **状态管理：** 使用 `st.session_state` 存储当前局已录入的多轮线索序列。

## 4. 给 AI 助手的开发指令 (Prompt Guidelines)

请严格遵循以下代码实现规范：
1.  **纯函数设计：** 将数学约束转化为纯函数，如 `check_divisibility(avg_grid, count)` 和 `check_price_bounds(avg_price, total_price, count)`。
2.  **浮点数安全：** 严禁在价格比对中直接使用 `==`，必须使用乘以 100 后的整数范围比对。平均格数转化为分数建议使用 Python 内置的 `fractions.Fraction`。
3.  **异常处理：** 如果求解器返回空集（表明录入线索有误产生数学矛盾），GUI 需要给予明显的红色 Warning 提示，而不是抛出系统崩溃异常。
4.  **性能优先：** 蒙特卡洛模拟部分必须使用 `numpy.random.choice` 的矢量化操作，不要使用原生 Python 的 `for` 循环逐个生成 10,000 个样本。
